## Build Flow

The recommended local command is:

```bash
make desktop-build
```

It runs:

1. `scripts/build-backend.sh`
2. `cd frontend && npm run tauri:build`

The backend sidecar is built with PyInstaller and copied to:

```text
frontend/src-tauri/binaries/devsynapse-backend-{target-triple}
```

Tauri then packages the React frontend, Rust shell and sidecar together.

Linux `.deb` and `.rpm` bundles include `frontend/src-tauri/scripts/linux-preremove.sh`
as a package pre-remove script. The hook stops the tray process and Python
backend sidecar before package files are removed, which avoids leaving a
running desktop build after uninstall. It intentionally does not delete runtime
config, databases, logs or repository files.

## Desktop Updater

The packaged desktop app includes a Tauri updater entry point in Settings.
Normal local builds keep it disabled unless the binary is compiled with a
public update-signing key:

```bash
export DEVSYNAPSE_UPDATER_PUBLIC_KEY="public key generated by tauri signer"
export DEVSYNAPSE_UPDATER_ENDPOINT="https://github.com/N1ghthill/devsynapse-ai/releases/latest/download/latest.json"
```

Signed updater artifacts require the matching private key at build time:

```bash
export TAURI_SIGNING_PRIVATE_KEY="$(cat "$HOME/.tauri/devsynapse-updater.key")"
export TAURI_SIGNING_PRIVATE_KEY_PASSWORD=""
make desktop-build-updates
```

Generate the signing key once and keep the private key out of the repository:

```bash
cd frontend
npm run tauri signer generate -- -w ~/.tauri/devsynapse-updater.key
```

After uploading updater bundles and their `.sig` files to a release, generate
the static manifest. Repeat `--platform`, `--url` and `--signature-file` once
per supported platform:

```bash
python3 scripts/generate-tauri-update-manifest.py \
  --version 0.6.2 \
  --platform linux-x86_64 \
  --url "https://github.com/N1ghthill/devsynapse-ai/releases/download/v0.6.2/DevSynapse_AI_0.6.2_amd64.deb" \
  --signature-file "frontend/src-tauri/target/release/bundle/deb/DevSynapse AI_0.6.2_amd64.deb.sig" \
  --platform windows-x86_64 \
  --url "https://github.com/N1ghthill/devsynapse-ai/releases/download/v0.6.2/DevSynapse_AI_0.6.2_x64-setup.exe" \
  --signature-file "frontend/src-tauri/target/release/bundle/nsis/DevSynapse AI_0.6.2_x64-setup.exe.sig" \
  --notes "Release notes" \
  --output latest.json
```

Upload `latest.json` as a release asset. The updater validates signatures before
installing an artifact.

## Cross-OS Rule

PyInstaller does not cross-compile the Python backend. Generate release artifacts
on the target operating system:

- Linux artifacts on Linux
- macOS artifacts on macOS
- Windows artifacts on Windows

`TAURI_TARGET_TRIPLE` or `CARGO_BUILD_TARGET` can select a Rust target triple,
but the Python sidecar still needs to be produced for the target OS.

If macOS links are shown, they should point only to artifacts produced by
target-OS CI jobs or manually validated target machines.

## Optional AppImage

The default Linux Tauri config builds `deb` and `rpm` only. AppImage can be tried
explicitly:

```bash
cd frontend
npm run tauri:build:appimage
```

Treat AppImage as experimental until `linuxdeploy` succeeds consistently in the
release environment.
