# Release v0.3.1

## Summary

`v0.3.1` is a hardening release for contributor reliability, CI coverage and local setup safety.
It keeps the v0.3.0 product surface intact while making installation, script execution and project-path handling portable across local machines and GitHub Actions runners.

## Highlights

### CI and verification

- GitHub Actions now runs backend lint, backend tests, shell syntax checks, ShellCheck, Python script compilation, frontend lint and frontend build.
- The CI workflow uses current GitHub Actions runtime versions and runs on pushes, pull requests, weekly schedule and manual dispatch.
- `make verify` now covers Python lint, backend tests, script checks, frontend lint and frontend build.
- Runtime dependencies and development/test dependencies are separated between `requirements.txt` and `requirements-dev.txt`, with checked-in lock constraints for reproducible installs.

### Installer and runtime scripts

- `scripts/install.sh` writes `.env` values safely without corrupting paths or keys containing shell-sensitive characters such as `&`.
- The installer now prompts for an admin password instead of silently relying on `admin/admin`.
- `scripts/uninstall.sh` passes ShellCheck and continues to support both keep-data and delete-data flows.
- `scripts/dev.py` now shuts down backend and frontend process groups cleanly on `SIGINT` and `SIGTERM`.

### Portability and project attribution

- Tests no longer depend on machine-specific absolute paths and now use repository-relative paths.
- `OpenCodeBridge` can receive project and allowed-directory context per instance, improving test isolation.
- `MemorySystem` can infer project attribution from persisted project records, not only process-level settings.

### Automated script coverage

- Added smoke tests for installer `.env` updates, alias creation, reinstall idempotence and uninstall cleanup choices.
- Added a process-group shutdown test for `scripts/dev.py`.

## Breaking changes

- None.

## Migration

For development environments, refresh dependencies and run the expanded verification gate:

```bash
make install-dev
cd frontend && npm install
cd ..
make verify
```

If you installed with an earlier script, rerun the installer to configure the admin password prompt and refreshed aliases:

```bash
bash scripts/install.sh
```
