# Release v0.3.2

## Summary

`v0.3.2` is a public-readiness patch for the local install and launch flow.
It keeps the product surface stable while tightening browser smoke coverage, dependency automation and launcher behavior.

## Highlights

### User flow hardening

- `devsynapse.sh` no longer prints a custom admin password from `.env`.
- `scripts/dev.py` also masks custom admin passwords in its startup output.
- The shell launcher now starts backend and frontend in process groups and terminates both cleanly on shutdown.
- Clean clone install, launch and stop flow was revalidated with disposable local state.

### CI and maintenance

- CI now includes a browser smoke job covering login, dashboard, settings save and admin navigation.
- Dependabot is configured for GitHub Actions, Python and frontend dependencies.
- Added a weekly/manual Python lock refresh workflow.
- Added a tag-driven GitHub Release workflow that reads `docs/releases/<tag>.md`.

## Breaking Changes

- None.

## Migration

Existing local installs can update normally:

```bash
git pull
bash scripts/install.sh
```

Then restart the launcher:

```bash
devsynapse
```
