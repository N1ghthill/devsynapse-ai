# Release v0.3.3

## Summary

`v0.3.3` is a post-`v0.3.2` release-readiness patch.
It tightens the local runtime model, Docker delivery, project administration,
authorization boundaries and command-execution telemetry.

## Highlights

### Runtime and delivery

- Runtime config, SQLite data and logs now live outside the source checkout by default through XDG paths or `DEVSYNAPSE_HOME`.
- `scripts/ensure_runtime_config.py` bootstraps runtime config for non-interactive setup flows.
- The installer, launcher and uninstaller now operate against the runtime config path instead of relying on a repo-local `.env`.
- The installer generates a strong JWT secret when the template/default value is unsafe and updates the configured admin user's password.
- The default Docker build now produces a FastAPI runtime image with the frontend production bundle included.
- Production frontend builds default to same-origin API calls when `VITE_API_URL` is unset.
- `.dockerignore` keeps local runtime artifacts, dependency directories and generated files out of the container build context.
- `nginx.conf` is now documented as an optional reverse-proxy baseline for the current FastAPI routes and `/chat/stream` SSE endpoint.
- CORS origins are configurable through `CORS_ALLOWED_ORIGINS`; wildcard origins no longer enable credentialed CORS.

### Authorization, projects and telemetry

- Chat, conversation, feedback and project routes now require authenticated users.
- Admins can list and register existing local project directories through `/admin/projects` and the Admin UI.
- Admin users show global registered-project mutation scope instead of editable per-user allowlists.
- Public `/projects` responses hide local project paths; `/admin/projects` remains the path-bearing management surface.
- Project-relative `read`, `edit` and `write` paths are normalized against the selected project before path validation.
- Mutating file and path-based bash commands must stay inside the resolved registered project.
- Command-level blocks and execution failures stay in command telemetry and no longer create false API `500` telemetry.
- API request telemetry middleware now attaches background logging tasks to the actual response object.

### Release process

- The release checklist now matches the current local gate: `make verify`, `make ui-smoke`, `pip check` and frontend audit.
- CI script compilation now includes `scripts/ensure_runtime_config.py`.
- Runtime, API and data-model documentation were refreshed to match the current contracts.

## Breaking Changes

- None.

## Migration

Existing local installs can update normally:

```bash
git pull
bash scripts/install.sh
```

Then restart the launcher:

```bash
devsynapse
```
