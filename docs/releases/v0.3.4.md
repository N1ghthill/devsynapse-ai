# Release v0.3.4

## Summary

`v0.3.4` adds a first-class update path for existing local installs.
Users no longer need to rerun the interactive installer just to move from one
published version to another.

## Highlights

### Update flow

- Added `scripts/update.sh` for non-interactive updates.
- Added `devsynapse update` through the launcher.
- Added the direct `update-devsynapse` alias during install.
- Added `make update` for maintainers and local contributors.

### Runtime safety

- The updater backs up existing runtime config, SQLite databases and log file when present.
- The updater preserves existing API key, admin password, repository path and runtime path settings.
- The updater can target the current branch, a specific branch or a published tag.
- The updater refreshes dependencies, guarantees runtime config, applies migrations, updates seeded users and rebuilds the frontend.

### Maintenance

- CI, `make script-check` and ShellCheck now include `scripts/update.sh`.
- Shell and development launcher banners now read the application version from `config/settings.py` instead of hardcoding it.

## Breaking Changes

- None.

## Migration

For users already on `v0.3.3` or earlier, the last manual update can still be:

```bash
git pull
bash scripts/install.sh
```

After that, future updates can use:

```bash
devsynapse update
```

or:

```bash
update-devsynapse
```

To pin a specific release:

```bash
devsynapse update --version v0.3.4
```
