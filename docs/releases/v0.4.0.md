# Release v0.4.0

## Summary

`v0.4.0` upgrades DevSynapse AI's DeepSeek integration from prompt/regex-driven
command extraction to native OpenAI-compatible tool calling, adds thinking-mode
support, and tightens the local-first security model for downloaded installs.

This release keeps the product direction intentionally local: users run the app
on their own machine, configure their own DeepSeek API key, and keep command
execution constrained by project-aware authorization.

## Highlights

### DeepSeek tool calling

- Added native DeepSeek tool calls with strict function definitions for `bash`, `read`, `glob`, `grep`, `edit` and `write`.
- Added thinking mode configuration through `DEEPSEEK_THINKING_ENABLED` and `DEEPSEEK_REASONING_EFFORT`.
- Changed the default model to `deepseek-v4-pro` and default base URL to `https://api.deepseek.com/beta`.
- Omitted `temperature` automatically when thinking mode is enabled.
- Kept regex command extraction as a fallback for degraded or legacy model responses.

### Chat and execution workflow

- Added streamed reasoning events to the frontend chat flow.
- Added execution result interpretation: after a confirmed command runs, the LLM can summarize the output in natural language.
- Added `interpretation` to `CommandExecutionResponse` and the frontend `CommandResult` type.
- Auto-executed read-only tool calls now replay only the executed tool call back to the LLM.
- Read-only auto-execution now uses a conservative bash allowlist instead of treating every user-authorized bash command as safe.

### Local-first security

- Default CORS origins are now limited to `http://127.0.0.1:5173` and `http://localhost:5173`.
- API startup logs a warning when the backend is bound to a non-loopback host.
- FastAPI startup/shutdown moved from deprecated `on_event` handlers to lifespan management.
- Tool-call conversion preserves escaped quotes, backslashes, tabs and newlines in `edit`/`write` arguments.
- Added a local security model and operator checklist for downloaded installs.
- Expanded tests for path traversal, symlink escapes, exposed-host warnings and conservative auto-execution.

## Breaking Changes

- The default DeepSeek model changed from `deepseek-chat` to `deepseek-v4-pro`.
- The default DeepSeek base URL changed from `https://api.deepseek.com/v1` to `https://api.deepseek.com/beta`.
- Default CORS behavior changed from wildcard to localhost-only frontend origins.

Existing local installs can override these values in runtime configuration if
they need the previous behavior.

## Migration

Update normally:

```bash
devsynapse update
```

or pin this release:

```bash
devsynapse update --version v0.4.0
```

Review runtime config after updating:

```env
DEEPSEEK_MODEL=deepseek-v4-pro
DEEPSEEK_BASE_URL=https://api.deepseek.com/beta
DEEPSEEK_THINKING_ENABLED=true
DEEPSEEK_REASONING_EFFORT=high
CORS_ALLOWED_ORIGINS=http://127.0.0.1:5173,http://localhost:5173
```

For network or container access, explicitly set trusted browser origins in
`CORS_ALLOWED_ORIGINS` and keep the API exposed only on trusted networks.
