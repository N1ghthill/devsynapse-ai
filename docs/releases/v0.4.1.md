# Release v0.4.1

## Summary

`v0.4.1` is a focused hotfix release after `v0.4.0`. It keeps the native
DeepSeek tool-calling direction, fixes the admin execution workflow so the
local operator can work productively, and tightens non-admin auto-execution
around file-content tools.

## Highlights

### Trusted admin execution

- Admin users are now treated as trusted local operators for command execution.
- Admins can run any supported OpenCode tool: `bash`, `read`, `glob`, `grep`,
  `edit` and `write`.
- Admin `bash` uses shell mode so pipelines, redirects and chained commands work.
- Admin file tools are not constrained to registered project roots or the normal
  allowed-directory list.
- Command telemetry and configured blacklist checks still apply, but this remains
  a local operator trust model, not a sandbox boundary.

### Safer non-admin auto-execution

- Non-admin chat auto-execution no longer runs file-content tools such as `read`,
  `grep` or `glob`.
- Non-admin users still get low-risk bash inspection auto-execution through the
  conservative allowlist.
- File-content commands are proposed for explicit confirmation instead of being
  sent to the LLM automatically.

### Tool-call parsing

- OpenCode command parsing now accepts escaped quotes in the primary command
  argument.
- Quoted shell snippets produced by DeepSeek tool calls are handled more reliably.

### Dependency maintenance

- Frontend dependencies were refreshed through Dependabot.
- Python runtime and development dependency manifests were refreshed through
  Dependabot, with lock constraints already updated in the repository.

## Platform Support

The supported installer/update path for this release is Linux on Debian/Ubuntu
or close `apt`-based derivatives. The scripts assume Bash, `python3`,
`python3-venv`, `npm` and Linux-style paths.

Native Windows setup is not validated for `v0.4.1`, and there is no PowerShell
or `.bat` installer. Windows users should use WSL2 with Ubuntu/Debian for the
supported installation flow. Manual native Windows usage may be possible because
the backend is Python and the frontend runs in a browser, but command execution,
aliases, path handling and update behavior should be treated as experimental.

## Release Compliance

- Application version: `0.4.1`
- Release tag: `v0.4.1`
- Supported installer target: Debian/Ubuntu-style Linux with `apt`
- Windows guidance: use WSL2 with Ubuntu/Debian; native Windows is unvalidated
- Runtime state: config, SQLite data and logs are outside the source checkout by
  default
- Security framing: trusted local operator model for admins, not kernel-level
  sandbox isolation
- Database migration impact: none for this release
- Release checklist: `make verify`, `make ui-smoke`, `pip check` and
  `npm audit --audit-level=high`

## Migration

Update normally:

```bash
devsynapse update
```

or pin this release:

```bash
devsynapse update --version v0.4.1
```

No database migration is required for this release.
