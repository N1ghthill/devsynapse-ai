# Release v0.5.0

## Summary

`v0.5.0` moves DevSynapse AI from a prompt-passing assistant toward a local
coding agent that can learn from outcomes. The release adds persistent
agent-learning memory, semantic task signatures, learned Flash/Pro routing and
validated Linux desktop packages.

## Highlights

### Local agent learning

- The agent now records route decisions, task signatures, selected models and
  outcome evidence locally in SQLite.
- User feedback and command execution outcomes can reinforce or adjust future
  model-routing preferences.
- Confident learned preferences can choose Flash or Pro for similar future
  tasks.
- Critical budget economy mode still takes precedence and keeps routing
  conservative.

### Smarter DeepSeek routing

- Tasks are profiled by type and semantic signature before routing.
- The prompt can include learned local patterns so the model receives useful
  project-specific operating context.
- Monitoring stats expose learned pattern counts and success/failure signals.

### Desktop production artifacts

- The Tauri desktop flow builds a PyInstaller backend sidecar and Linux desktop
  packages.
- Linux `.deb` and `.rpm` artifacts are validated for this release on the local
  Linux build environment.
- AppImage remains opt-in because `linuxdeploy` is environment-sensitive.
- macOS and Windows packaging are configured but still require target-OS
  validation before public download links.

## Platform Support

The supported installer/update path for this release is Linux on Debian/Ubuntu
or close `apt`-based derivatives. The scripts assume Bash, `python3`,
`python3-venv`, `npm` and Linux-style paths.

Native Windows setup is not validated for `v0.5.0`, and there is no PowerShell
or `.bat` installer. Windows users should use WSL2 with Ubuntu/Debian for the
supported installation flow. Manual native Windows usage may be possible because
the backend is Python and the frontend runs in a browser, but command execution,
aliases, path handling and update behavior should be treated as experimental.

## Release Compliance

- Application version: `0.5.0`
- Release tag: `v0.5.0`
- Supported installer target: Debian/Ubuntu-style Linux with `apt`
- Windows guidance: use WSL2 with Ubuntu/Debian; native Windows is unvalidated
- Runtime state: config, SQLite data and logs are outside the source checkout by
  default
- Security framing: trusted local operator model for admins, not kernel-level
  sandbox isolation
- Database migration impact: migration `9` creates `agent_learning` and
  `agent_route_decisions`
- Release checklist: `make verify`, `make desktop-build`, `make ui-smoke`,
  `pip check` and `npm audit --audit-level=high`

## Migration

Update normally:

```bash
devsynapse update
```

or pin this release:

```bash
devsynapse update --version v0.5.0
```

The updater applies SQLite migrations automatically. Migration `9` adds local
agent-learning tables; it does not remove existing conversation, command,
settings or usage data.
