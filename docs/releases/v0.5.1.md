# Release v0.5.1

## Summary

`v0.5.1` is the launch packaging release for DevSynapse AI. It keeps the
`v0.5.0` agent-learning baseline and adds validated Windows desktop artifacts
to the signed GitHub release flow.

## Highlights

### Windows desktop build

- GitHub Actions now builds the Windows desktop app on `windows-latest`.
- The Python backend sidecar is generated on Windows with PyInstaller as
  `devsynapse-backend-x86_64-pc-windows-msvc.exe`.
- Tauri produces the Windows installer artifact and matching updater signature.
- The packaged sidecar is smoke-checked with `--help` during the Windows job.

### Combined updater manifest

- Release publishing now waits for both Linux and Windows desktop jobs.
- `latest.json` includes:
  - `linux-x86_64`
  - `windows-x86_64`
- Release assets include signed installer artifacts and `SHA256SUMS`.

### Existing v0.5 agent layer

- Persistent local agent learning remains enabled.
- Semantic task signatures continue to guide Flash/Pro routing.
- Critical budget economy mode still takes precedence over learned preferences.

## Platform Support

The supported desktop downloads for this release are:

- Linux x86_64 `.deb`
- Linux x86_64 `.rpm`
- Windows x86_64 Tauri installer

The supported shell installer/update path remains Debian/Ubuntu-style Linux.
Native Windows shell scripts are not part of the support surface; Windows users
should use the desktop installer for the validated Windows path.

macOS packaging remains configured but not yet validated on a macOS runner.

## Release Compliance

- Application version: `0.5.1`
- Release tag: `v0.5.1`
- Supported desktop targets: Linux x86_64 and Windows x86_64
- Supported shell installer target: Debian/Ubuntu-style Linux with `apt`
- Runtime state: config, SQLite data and logs are outside the source checkout by
  default
- Security framing: trusted local operator model for admins, not kernel-level
  sandbox isolation
- Database migration impact: no new migration beyond `v0.5.0`; migration `9`
  creates `agent_learning` and `agent_route_decisions`
- Release workflow: Linux build, Windows build, combined publish job

## Migration

Update normally:

```bash
devsynapse update
```

or pin this release:

```bash
devsynapse update --version v0.5.1
```

The updater applies SQLite migrations automatically. Users already on `v0.5.0`
do not need an additional database migration for `v0.5.1`.
