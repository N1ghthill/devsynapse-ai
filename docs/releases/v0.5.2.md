# Release v0.5.2

## Summary

`v0.5.2` is a production hardening release for the packaged desktop app. It
keeps the `v0.5.1` Linux and Windows distribution surface and moves interactive
runtime setup into the app so Tauri installs no longer depend on the legacy
shell installer flow.

## Highlights

### First-run desktop setup

- The app now checks `/bootstrap/status` before normal authenticated use.
- New installs collect:
  - local admin password
  - DeepSeek API key
  - workspace root
  - default repository folder
- `/bootstrap/complete` registers discovered Git projects and refreshes the
  command bridge scope for the current process.

### Runtime configuration

- Runtime `.env` creation now happens before application settings are loaded.
- DeepSeek API key changes made through Settings are persisted to the runtime
  config.
- Existing shell installs keep using the same runtime config/data/log locations.

### Updater safety

- `scripts/update.sh` still backs up runtime state, refreshes dependencies,
  applies migrations and rebuilds the frontend.
- The updater no longer overwrites an existing admin password while ensuring
  missing default users.
- Existing installs with configured API key and workspace do not re-enter
  onboarding after update.

### Knowledge memory

- Project-scoped procedural memories, Markdown-backed skills and learning nudge
  events are persisted through SQLite migration `10`.
- Non-admin memory writes require project mutation permission; global memory
  writes remain admin-only because saved knowledge is injected into future
  prompts.

## Platform Support

The supported desktop downloads remain:

- Linux x86_64 `.deb`
- Linux x86_64 `.rpm`
- Windows x86_64 Tauri installer

The supported shell installer/update path remains Debian/Ubuntu-style Linux.
macOS packaging remains configured but unvalidated.

## Release Compliance

- Application version: `0.5.2`
- Release tag: `v0.5.2`
- Supported desktop targets: Linux x86_64 and Windows x86_64
- Supported shell installer target: Debian/Ubuntu-style Linux with `apt`
- Runtime state: config, SQLite data and logs remain outside the source checkout
- Security framing: trusted local operator model for admins, not kernel-level
  sandbox isolation
- Database migration impact: migration `10` adds `project_memories`, `skills`,
  `skill_activations` and `learning_nudge_events`

## Migration

Update normally:

```bash
devsynapse update
```

or pin this release:

```bash
devsynapse update --version v0.5.2
```

Existing configured installs should keep their runtime config. Packaged desktop
users missing API key or workspace configuration will be guided through Setup
inside the app.
