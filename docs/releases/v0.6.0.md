# Release v0.6.0

## Summary

`v0.6.0` is a project workflow release for DevSynapse AI. It makes the chat
session explicitly project-aware, adds project creation/selection controls to
the app, and tightens local command execution so write operations stay inside
the selected working repository while read-only cross-repo inspection remains
available.

## Highlights

### Project-scoped chat workflow

- Chat conversations now persist the selected project and reject attempts to
  switch to a different project inside the same conversation.
- Selecting an existing project or creating a new one from the chat interface
  opens a matching project conversation context.
- The UI now exposes project selection and admin-only project creation from the
  chat workflow instead of relying only on inferred project names.

### Repository working directory safety

- Mutating command bridge actions are constrained to the selected project path.
- Write, edit and mutating shell operations outside the selected project return
  a `project_scope_mismatch` result instead of touching another repository.
- Read-only references remain allowed, so the assistant can inspect another
  repository for context while continuing to work inside the designated project.

### Development flow reliability

- Streaming chat handling is more tolerant of abrupt stream completion.
- Runtime bootstrap refreshes cached settings after setup changes, keeping the
  configured workspace and repository roots available without a restart.
- Tauri updater manifest generation is now covered by tests and wired into the
  release automation.

## Platform Support

The supported desktop downloads remain:

- Linux x86_64 `.deb`
- Linux x86_64 `.rpm`
- Windows x86_64 Tauri installer

The supported shell installer/update path remains Debian/Ubuntu-style Linux.
macOS packaging remains configured but unvalidated.

## Release Compliance

- Application version: `0.6.0`
- Release tag: `v0.6.0`
- Supported desktop targets: Linux x86_64 and Windows x86_64
- Supported shell installer target: Debian/Ubuntu-style Linux with `apt`
- Runtime state: config, SQLite data and logs remain outside the source checkout
- Security framing: trusted local operator model for admins, not kernel-level
  sandbox isolation
- Database migration impact: none

## Migration

Update normally:

```bash
devsynapse update
```

or pin this release:

```bash
devsynapse update --version v0.6.0
```

Existing configured installs should keep their runtime config. Existing chats
without a persisted project keep their previous behavior until a project is
selected for the conversation.
