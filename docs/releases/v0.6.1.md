# Release v0.6.1

## Summary

`v0.6.1` is a patch release for project registry hygiene and packaged desktop
uninstall reliability.

## Highlights

### Project Registry Cleanup

- User-facing project lists now hide registered projects whose local directory
  no longer exists.
- Admin project management shows active and stale registry rows with
  `path_exists` status.
- Admins can remove a project registry entry without deleting files from disk.
- Re-registering a stale project name reuses the stale registry row instead of
  failing as a duplicate.
- Project lookup and prompt context ignore stale project paths.

### Desktop Uninstall Reliability

- Linux desktop `.deb` and `.rpm` packages now include a pre-remove hook that
  stops the tray app and backend sidecar before package files are removed.
- The uninstall hook intentionally preserves runtime config, SQLite databases,
  logs and user repositories.

## Platform Support

The supported desktop downloads remain:

- Linux x86_64 `.deb`
- Linux x86_64 `.rpm`
- Windows x86_64 Tauri installer

The supported shell installer/update path remains Debian/Ubuntu-style Linux.
macOS packaging remains configured but unvalidated.

## Release Compliance

- Application version: `0.6.1`
- Release tag: `v0.6.1`
- Supported desktop targets: Linux x86_64 and Windows x86_64
- Supported shell installer target: Debian/Ubuntu-style Linux with `apt`
- Runtime state: config, SQLite data and logs remain outside the source checkout
- Database migration impact: none

## Migration

Update normally:

```bash
devsynapse update
```

or pin this release:

```bash
devsynapse update --version v0.6.1
```

Existing project registry rows are not deleted automatically. Stale rows are
hidden from normal project selection and can be removed from Admin > Projects.
